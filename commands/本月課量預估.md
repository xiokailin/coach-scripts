# 本月課量預估

**觸發：** `/本月課量預估`
**Tab：** `printf '\033]0;CourseCount-MM\007'`

---

## 使用前設定

| 設定項目 | 說明 | 填入你的值 |
|---------|------|-----------|
| 業績回報路徑 | 每日業績報表 .txt 存放位置 | 例：`~/Downloads/coach-scripts/業績回報` |
| gws 路徑 | gws 執行檔位置 | 例：`/opt/homebrew/bin/gws`（`which gws` 確認） |
| 上課行程關鍵字 | 篩選行事曆用的標記 | 例：`一對一`、`一對二` |

---

## 執行流程

### 1. 詢問排除學員

```
需要排除哪幾位學員的未來課程？（可留空，直接 Enter 跳過）
```

### 2. 自動取得昨日已上堂數

讀取昨天的業績報表，grep「已上堂數」取出數字作為「已完成堂數」。若檔案不存在，改問使用者。

### 3. 查詢本月行事曆

用 gws 查詢本月所有行程，篩選 summary 含「一對一」或「一對二」的行程。

### 4. 計算

只計算**今日起的未來行程**：
- summary 含「請假」→ 跳過
- summary 含任一排除學員姓名 → 跳過，記錄到排除明細

### 5. 輸出

```
【本月課量預估 — YYYY/M】

截至昨日已上：N 堂（來源：業績報表 M/D）

剩餘行程（M/D 起）：
M/DD  N 堂  [若有扣除說明]
...
合計剩餘：N 堂

月底預估：N + N = N 堂
──────────────────────
【排除明細】
學員：N 堂（日期列舉）
合計排除 N 堂 → 若不排除：N 堂
```

---

## 規則

- 一對二 = 1 堂（不拆成 2）
- 排除只影響未來行程，不動行事曆、不修改已上堂數
- 備註含「改時間」「last」「請教」仍計入

---

## 附加輸出：請假率試算 + 薪資估算

主要輸出後，自動附上：

```
【請假率試算 + 薪資估算】
| 請假率 | 月底合計 | 預估薪資 |
|--------|---------|---------|
| 10%    | N 堂    | ~$XX,XXX |
| 15%    | N 堂    | ~$XX,XXX |
| 20%    | N 堂    | ~$XX,XXX |
```

薪資估算以「薪資試算」指令的設定值為準。
