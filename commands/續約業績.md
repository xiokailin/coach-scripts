# 生成下個月預計續約業績

**觸發：** `生成下個月續約` 或 `/續約業績`
**Tab：** `printf '\033]0;Renewal-MM\007'`

月底執行，計算下個月會消耗完課堂數的學員，更新續約追蹤 Sheet。

---

## 使用前設定

| 設定項目 | 說明 | 填入你的值 |
|---------|------|-----------|
| 教練姓名 | 你在 Sheet A 欄的名字 | [填入] |
| 續約追蹤 Sheet ID | Google Sheets 網址中的 ID 字串 | [填入] |

**月份分頁 GID 對照（每月底建新分頁時記錄一次）：**

| 分頁名稱 | GID |
|---------|-----|
| 1月預計續約 | [GID] |
| 2月預計續約 | [GID] |
| 3月預計續約 | [GID] |
| 4月預計續約 | [GID] |
| 5月預計續約 | [GID] |
| 6月預計續約 | [GID] |
| 7月預計續約 | [GID] |
| ... | ... |

---

## 資料來源優先順序

1. **student_*.md 的「最新合約」區塊**（最準確）
2. **student_*.md 的「剩餘堂數追蹤」區塊**（有「預計完課」欄位）
3. **續約追蹤 Sheet 歷史** — 讀當月分頁 A=[教練姓名] 且 F="TRUE" 的列，取日期最新一筆

---

## 執行步驟

**第一步：確認目標月份**
根據今日日期自動判斷下個月（例：今日 6/30 → 下個月 = 7 月）

**第二步：收集學員合約資料**
讀取所有 `student_*.md`，依序取：
- 「最新合約」區塊 → 合約日期、堂數、金額、頻率
- 「剩餘堂數追蹤」區塊 → 預計完課日

若兩者都有，以「剩餘堂數追蹤」的「預計完課」為準（反映實際進度）。

**第三步：篩選下個月完課者**
預計完課日落在目標月份 → 納入清單；已畢業/停課學員跳過。

**第四步：顯示草稿等待確認**

```
【X月預計續約業績草稿 — [教練姓名]】

學員名稱 | 堂數 | 金額      | 預計日期 | 頻率
-------- | ---- | --------- | -------- | ----
學員A    | 8   | NT$XX,XXX | X/XX     | 1x/w
...

合計預計：NT$XXX,XXX
```

對有「?」的欄位：標示「合約無記錄，請確認」，等使用者填入。

**第五步：使用者確認後，寫入 Sheet**

1. 讀取目標月份分頁（如「7月預計續約」）
2. 搜尋 A 欄為「[教練姓名]」的第一列 → 確定起始 row
3. 用 `sheets_values_update` 逐行寫入（F 欄不填，留空）：
   `A=[教練姓名], B=學員姓名, C=堂數, D=NT$金額, E=YYYY/M/D`
4. 回傳 Sheet 網址

---

## 寫入格式

| 欄 | 內容 |
|----|------|
| A | 教練姓名 |
| B | 學員姓名（一對二合併，如「學員A/學員B」） |
| C | 預計堂數 |
| D | NT$金額 |
| E | 預計完課日（YYYY/M/D） |
| F | **留空**（成交後店長手動改 TRUE） |
| G–H | 空白（待實際成交後手動填入） |

---

## 規則

- F 欄不填：不自動寫入 FALSE，留空即可
- 一對二學員合併寫在 B 欄
- 已畢業學員直接略過
- 若合約金額無記錄，草稿標示「?」等使用者補充後再確認
- 分頁不存在 → 提示：「X月分頁尚未建立，請店長建立後再執行」
