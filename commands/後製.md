# 影片後製流程

使用者拍攝完、剪映節奏剪輯完成後，觸發此指令。

**觸發方式：** `/後製 [.srt 檔案路徑]`
若未提供路徑，詢問使用者 .srt 在哪裡。

---

## 執行步驟（全程不需確認，直接完成）

### Step 1：讀取 .srt

讀取使用者提供的 .srt 檔案。

### Step 2：簡體 → 繁體轉換

- 將所有簡體字直接轉換為繁體中文
- 保留原始時間軸，格式完整維持 SRT 規範
- 修正剪映常見辨識錯誤（同音字、斷句錯誤）
- 專業術語維持英文（ACSM、RPE、Beta blocker、ACEI/ARB 等）

### Step 3：文獻配對

分析字幕內容，找出需要佐證的專業知識點（每個明確的醫學或運動科學論點配一篇文獻）。

文獻格式：
- 優先使用 meta-analysis 或 systematic review
- 2015 年後優先，2020 年後更佳
- 來源：PubMed、ACSM 期刊、JAMA、NEJM、Lancet 等
- 顯示格式：`年份 發表在[期刊縮寫]的研究：[論文標題關鍵詞]`

將文獻以「額外 SRT 條目」方式附加在字幕檔案尾端，格式對齊使用者現有習慣：
```
[序號]
[文獻出現的開始時間] --> [結束時間]
[文獻描述文字]
```
時間軸設定：對應到字幕中提及該知識點的時間點，顯示約 3–4 秒。

### Step 4：輸出繁體 .srt 檔案

- 儲存路徑：與原始 .srt 同資料夾，檔名加上 `_繁體` 後綴
- 例如：`3月22日.srt` → `3月22日_繁體.srt`

### Step 5：IG 說明文字 + Hashtag

**語氣原則（與 `/腳本` 一致）：**
不批評或否定他人建議，使用「肯定出發點 → 補充進階需求」結構。禁用否定句型（「其實是錯的」「這樣做沒有用」等）。

**說明文字格式：**
```
【[主題標題]】—（上/下集，若適用）

[2–3 句開場白，點出觀眾痛點或常見迷思]

[3–5 個重點條列，對應影片內容]

[1 句結語 call-to-action]

#hashtag1 #hashtag2 ...
```

**Hashtag 原則：**
- 10–15 個
- 組合：主題標籤（退化性關節炎、高血壓）+ 族群（銀髮族、特殊族群）+ 專業（職能治療師、ACSM）+ 通用（運動、健康）
- 繁體中文為主，1–2 個英文

---

## 最終輸出格式

對話中顯示：
1. 繁體 .srt 已儲存路徑
2. 文獻配對清單（知識點 → 文獻）
3. IG 說明文字全文
